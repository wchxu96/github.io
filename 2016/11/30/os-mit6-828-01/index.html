正文看书笔记

The job of an operating system is to share a computer among multiple programs
and to provide a more useful set of services than the hardware alone supports.
这一句写出了os的用途，是为了在多任务的调度方面提供解决方案，同时提供了比硬件接口更成熟更可用的服务
The
operating system manages and abstracts the low-level hardware, so that, for example, a
word processor need not concern itself with which type of disk hardware is being
used. It also multiplexes the hardware, allowing many programs to share the computer
and run (or appear to run) at the same time. Finally, operating systems provide con-
trolled ways for programs to interact, so that they can share data or work together.
操作系统抽象管理了低层次的硬件服务，因此程序不用在意自己在使用哪些硬件，应用程序只是在user-mode调用os提供的api而已
同时，操作系统更好的分配了cpu，io等资源使程序看上去像是同时运行的即使在单cpu的情况下，ipc(进程间数据共享也是一个重要话题)
Each running program, called a
process, has memory containing instructions, data, and a stack.
每一个进程都存在指令，数据，栈区存储在内存中
The instructions im-
plement the program’s computation. The data are the variables on which the computa-
tion acts. The stack organizes the program’s procedure calls.
写的明白!指令区负责执行实现计算过程，数据区负责提供程序所需的数据,栈区负责过程调用
When a process needs to invoke a kernel service, it invokes a procedure call in
the operating system interface. Such a procedure is called a system call. The system
call enters the kernel; the kernel performs the service and returns. Thus a process al-
ternates between executing in user space and kernel space
系统调用　用户空间　内核空间
An xv6 process consists of user-space memory (instructions, data, and stack) and
per-process state private to the kernel. Xv6 can time-share processes: it transparently
switches the available CPUs among the set of processes waiting to execute. When a
process is not executing, xv6 saves its CPU registers, restoring them when it next runs
the process. The kernel associates a process identifier, or pid, with each process.
进程的信息包括两部分，一部分是存在于用户空间的内存中，另一部分只有内核有权查看
xv6是分时系统，当一个进程不执行时，xv6会存储它的cpu寄存器，运行其他进程，然后恢复寄存器内容重新运行
pid的概念
A process may create a new process using the fork system call. Fork creates a
new process, called the child process, with exactly the same memory contents as the
calling process, called the parent process. Fork returns in both the parent and the
child. In the parent, fork returns the child’s pid; in the child, it returns zero. For ex-
ample, consider the following program fragment:
父进程，子进程　fork function

很简单的fork程序，学习wait和exit系统调用的用法

The exit system call causes the calling process to stop executing and to release re-sources such as memory and open files. The wait system call returns the pid of an
exited child of the current process; if none of the caller’s children has exited, wait
waits for one to do so.
wait是一个阻塞函数，当子进程不exit,它便一直阻塞，否则返回子进程pid
Note that the parent and child were executing with different memory and different
registers: changing a variable in one does not affect the other.
与线程区别，不同进程用不同的内存
The exec system call replaces the calling process’s memory with a new memory
image loaded from a file stored in the file system

exec系统调用，将执行进程的内存空间替换为另一个从文件系统中读取出来的内存映像.
The file must have a particular for-
mat, which specifies which part of the file holds instructions, which part is data, at
which instruction to start, etc. xv6 uses the ELF format, which Chapter 2 discusses in
more detail. When exec succeeds, it does not return to the calling program; instead,
the instructions loaded from the file start executing at the entry point declared in the
ELF header.
当exec调用成功后并不会回到原来的调用进程中
